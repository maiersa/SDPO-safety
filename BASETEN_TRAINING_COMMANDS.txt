SDPO METRIC CHEAT SHEET
================================================================================

Watch these for constitutional SDPO:
- actor/pg_loss                     # main optimization signal; should stay finite and non-zero
- actor/grad_norm                   # should stay finite; persistent spikes or zeros are bad
- actor/entropy                     # helps spot collapse/overconfidence
- self_distillation/reward_bypass   # should stay 1.0 in constitutional SDPO
- self_distillation/reprompt_sample_fraction      # should stay near 1.0 here
- self_distillation/constitution_prompt_fraction  # should stay 1.0 here
- self_distillation/empty_target_batch            # should stay 0.0
- training/rollout_actor_probs_pearson_corr       # should stay very close to 1.0
- rollout_corr/*                    # rollout consistency checks

Expected to stay ~0 in this SDPO setup:
- critic/rewards/*
- critic/advantages/*
- critic/returns/*

POST-TRAIN BULLSHITBENCH EVALUATION
================================================================================

# BullshitBench nonsense set is prepared by scripts/download_bullshitbench.sh.
# Evaluate checkpoints on nonsense prompts only.

# 1) Set the model/checkpoint path to evaluate.
MODEL_TO_EVAL=<PATH_TO_TRAINED_MODEL_OR_ACTOR_CKPT>
OUT_JSON=/dlabscratch1/${USER}/output/bullshitbench_eval_$(date +%Y%m%d_%H%M%S).json

# 2) Run evaluation with OpenAI constitutional judge scoring.
MODEL_TO_EVAL=$MODEL_TO_EVAL OUT_JSON=$OUT_JSON OPENAI_API_KEY=$OPENAI_API_KEY python - << 'PY'
import json
import os
from statistics import mean

import torch
from transformers import AutoModelForCausalLM, AutoTokenizer

from verl.utils.reward_score.feedback.constitution_judge import call_judge_api

model_path = os.environ.get("MODEL_TO_EVAL")
out_json = os.environ.get("OUT_JSON", "bullshitbench_eval.json")
data_path = "datasets/bullshitbench/nonsense_questions.json"

if not model_path or model_path.startswith("<PATH_TO"):
	raise SystemExit("Set MODEL_TO_EVAL to a real checkpoint/model path before running.")

with open(data_path, "r") as f:
	data = json.load(f)

tokenizer = AutoTokenizer.from_pretrained(model_path, trust_remote_code=True)
model = AutoModelForCausalLM.from_pretrained(
	model_path,
	trust_remote_code=True,
	torch_dtype=torch.bfloat16 if torch.cuda.is_available() else torch.float32,
	device_map="auto",
)

results = []
for i, item in enumerate(data):
	q = item.get("question") or item.get("prompt") or ""
	if not q:
		continue
	inputs = tokenizer(q, return_tensors="pt").to(model.device)
	with torch.no_grad():
		out = model.generate(
			**inputs,
			max_new_tokens=256,
			temperature=0.7,
			top_p=0.95,
			do_sample=True,
		)
	text = tokenizer.decode(out[0], skip_special_tokens=True)
	resp = text[len(q):].strip() if text.startswith(q) else text.strip()
	judge = call_judge_api(q, resp)
	results.append({
		"id": item.get("id", i),
		"correctness": judge["correctness"],
		"tone": judge["tone"],
		"combined_score": judge["combined_score"],
		"question": q,
		"response": resp,
	})
	if (i + 1) % 10 == 0:
		print(f"Scored {i + 1}/{len(data)} prompts")

summary = {
	"model_path": model_path,
	"num_scored": len(results),
	"mean_correctness": mean(r["correctness"] for r in results) if results else 0.0,
	"mean_tone": mean(r["tone"] for r in results) if results else 0.0,
	"mean_combined": mean(r["combined_score"] for r in results) if results else 0.0,
}

with open(out_json, "w") as f:
	json.dump({"summary": summary, "results": results}, f, indent=2)

print("\nBullshitBench evaluation summary:")
for k, v in summary.items():
	print(f"  {k}: {v}")
print(f"\nDetailed results: {out_json}")
PY


NOTES
================================================================================

- These direct rs commands call runai_grpo_worker.sh or runai_sdpo_worker.sh directly.
- They do not require run_runai_* wrappers, though wrappers remain available.
- Keep TRAINER_GPUS_PER_NODE aligned with the --gpu value for multi-GPU jobs.
- If WANDB_ENTITY is omitted, training/verl_training.sh defaults to offline wandb mode.
- For GRPO, judge failures are logged with running failure counts in constitution_judge.py.
