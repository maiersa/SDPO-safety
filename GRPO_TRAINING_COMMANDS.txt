================================================================================

COMMON PATHS
================================================================================

REPO_DIR=/dlabscratch1/${USER}/projects/SDPO-safety
CONDA_ENV=/mnt/dlabscratch1/samaier/conda-envs/sdpo-grpo
LOG_DIR=/dlabscratch1/${USER}/output

GRPO: DIRECT RS COMMAND (4 GPUS)
================================================================================

rs grpo-${USER}-tooluse --gpu 4.0 --cpu 16 --memory 120G --node-pools h100 --large-shm -- env \
USER=$USER \
CONDA_ENV=/mnt/dlabscratch1/samaier/conda-envs/sdpo-grpo \
CONFIG_NAME=baseline_grpo \
DATA_PATH=datasets/tooluse \
SUFFIX=tooluse \
REPO_DIR=/dlabscratch1/${USER}/projects/SDPO-safety \
LOG_DIR=/dlabscratch1/${USER}/output \
CKPT_DIR=/dlabscratch1/${USER}/checkpoints \
TRAIN_BATCH_SIZE=32 \
ROLLOUT_BATCH_SIZE=8 \
MINI_BATCH_SIZE=8 \
LR=1e-5 \
MODEL_PATH=Qwen/Qwen2.5-7B-Instruct \
TRAINER_GPUS_PER_NODE=4 \
WANDB_ENTITY=samaier-epfl \
bash /dlabscratch1/${USER}/projects/SDPO-safety/runai_grpo_worker.sh

NOTES
================================================================================

- These direct rs commands call runai_grpo_worker.sh or runai_sdpo_worker.sh directly.
- They do not use run_rs_grpo_submit.sh unless you explicitly run that helper script.
- Keep TRAINER_GPUS_PER_NODE aligned with the --gpu value for multi-GPU jobs.
- If WANDB_ENTITY is omitted, training/verl_training.sh defaults to offline wandb mode.
